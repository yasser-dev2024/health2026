# خطة بناء مشروع "صيف وصحة - مساعد"

## 1. وصف المشروع
منصة Django عربية باتجاه RTL تعمل بـ Server-Side Rendering لخدمة زوار منطقة عسير. تعرض فعاليات صحية، مواقع قريبة، رسائل توعوية، مكتبة تحميلات، خطة صحية يومية، مساعدا توعويا بالكلمات المفتاحية، جواز صحة بالنقاط والأختام، مسارات QR، وتحدي بطل الصحة.

## 2. الهدف من النظام
تقديم تجربة صحية سياحية بسيطة وآمنة للزائر، مع لوحة إدارة مخصصة لإدارة المحتوى والتقارير والميزات والصلاحيات دون الاعتماد على واجهة JavaScript أساسية.

## 3. الصفحات المطلوبة
الصفحات العامة: الرئيسية، الرحلة، الخطة، الفعاليات، الجواز، التحميلات، الرسائل، الدكتور مساعد، المواقع القريبة، تحدي بطل الصحة، مسارات QR للمواقع والعناصر.  
الإدارة: تسجيل الدخول، لوحة المؤشرات، إدارة QR، وروابط إدارة بقية الكيانات من اللوحة المخصصة ولوحة Django الرسمية.

## 4. التطبيقات المقترحة
- `core`: الصفحة الرئيسية، الإعدادات العامة، الأزرار السريعة، الميزات، أدوات الجلسة والتحقق.
- `journey`: نموذج الرحلة وخدمة بناء الخطة الصحية اليومية.
- `events`: الفعاليات الصحية.
- `locations`: المواقع الصحية والمهمة.
- `awareness`: الرسائل التوعوية ومكتبة التحميلات.
- `assistant`: الدكتور مساعد، الأسئلة، الكلمات المفتاحية، وسجل الاستخدام.
- `passport`: جواز صحة عسير، الأختام، الإنجازات، والنقاط.
- `qr`: مواقع QR، عناصر QR، التسجيل والتقارير.
- `hero`: تحدي بطل الصحة والنتائج.
- `dashboard`: لوحة الإدارة المخصصة وسجل العمليات.

## 5. مسؤولية كل تطبيق
كل تطبيق يملك نماذجه وقوالبه ومساراته. المنطق التجاري يوضع في `services.py` عند الحاجة، والنماذج في `forms.py` عند وجود إدخال من المستخدم أو الإدارة.

## 6. النماذج المطلوبة
`HomeConfig`, `HomeQuickButton`, `HealthEvent`, `HealthLocation`, `AwarenessContent`, `AwarenessMessage`, `KeywordAnswer`, `DoctorAssistantQuestion`, `PassportStamp`, `VisitorPassport`, `PassportAchievement`, `QrLocation`, `QrItem`, `QrScan`, `QrVisit`, `QrLocationVisit`, `HealthHeroQuestion`, `HealthHeroEntry`, `FeatureFlag`, `OperationLog`, `GeneralSettings`, `SmartEntryConfig`.

## 7. مسارات URL الرئيسية
`/`, `/journey/`, `/plan/`, `/events/`, `/passport/`, `/downloads/`, `/messages/`, `/assistant/`, `/nearby/`, `/hero/`, `/qr/<slug>/`, `/qr-item/<id>/`, `/admin/login/`, `/admin/`, `/admin/qr-locations/`, `/django-admin/`.

## 8. هيكلة Templates
القوالب العامة في `templates/base.html`, `templates/header.html`, `templates/footer.html`, `templates/home.html`.  
قوالب التطبيقات داخل مجلدات: `core`, `journey`, `events`, `locations`, `awareness`, `assistant`, `passport`, `qr`, `hero`, `dashboard`.

## 9. هيكلة Static
`static/css/identity.css` هو ملف الهوية الوحيد. المجلدات: `static/css`, `static/js`, `static/img`, `static/vendor`.

## 10. ملف الهوية البصرية
يعرف الألوان: `#15508A`, `#2FA9E0`, `#1691D0`, `#283A83`, `#057590`, `#A09EA9`, `#FFFFFF`, `#F4FAFC`، بالإضافة إلى الخطوط، المسافات، الأزرار، النماذج، البطاقات، الجداول، التنبيهات، الشارات، حالات hover/focus/disabled، والظلال.

## 11. منطق الخطة الصحية
تخزن إجابات الزائر في Session، وتنشئ `visitor_id` عشوائيا. الخدمة تراعي الوجهة، العمر، فرد/عائلة، الغرض، والحالة الصحية. الحالات العاجلة توجه إلى 997 و937، وكبار السن يحصلون على مسار أخف، والعائلات على فعاليات مناسبة.

## 12. منطق الدكتور مساعد
تطبيع عربي للنص: توحيد الهمزات، تحويل التاء المربوطة، إزالة التشكيل، توحيد المسافات. يتم البحث في `KeywordAnswer` و`DoctorAssistantQuestion`. كلمات الطوارئ تعرض رسالة واضحة تتضمن 997 و937 وتسجل الاستخدام.

## 13. منطق QR
كل مسح يسجل `visitor_id`, النوع، المسار، الوقت، والعنصر المرتبط. `QrItem` قد يمنح ختما، و`QrLocation` يسجل زيارة موقع. الإحصاءات داخل قاعدة البيانات. توليد PNG يتم عبر مكتبة `qrcode`.

## 14. منطق الجواز والنقاط
الجواز يعتمد على `visitor_id`. يمنع تكرار نفس الختم لنفس الزائر، وتضاف النقاط حسب الختم أو النشاط. الإنجازات تفتح عند بلوغ عدد أختام أو نقاط محددة.

## 15. منطق لوحة الإدارة
لوحة SSR محمية بتسجيل الدخول، تعرض مؤشرات المحتوى والزيارات، روابط إدارة الكيانات، تقرير QR، وإدارة QR locations. التحرير التفصيلي متاح عبر Django admin الرسمي على `/django-admin/`.

## 16. الصلاحيات
الأدوار: `owner`, `editor`, `viewer` عبر مجموعات Django. صفحات الإدارة تتطلب تسجيل الدخول، وتستخدم CSRF وجلسات HttpOnly. عمليات الإدارة المهمة تسجل في `OperationLog`.

## 17. البيانات الأولية المطلوبة
بيانات نموذجية للصفحة الرئيسية، الأزرار، الفعاليات، المواقع، الرسائل، التحميلات، أسئلة الدكتور مساعد، الكلمات المفتاحية، الأختام، الإنجازات، مواقع QR، عناصر QR، أسئلة التحدي، الإعدادات العامة، وSmart Entry.

## 18. خطة التنفيذ خطوة بخطوة
1. إنشاء التطبيقات بأمر `python manage.py startapp`.
2. ضبط settings للمسارات، القوالب، static، auth، وقاعدة البيانات.
3. تنفيذ النماذج والعلاقات.
4. تنفيذ الخدمات: visitor/session، الخطة، المساعد، QR، الجواز، التحدي.
5. تنفيذ النماذج Forms.
6. تنفيذ views وurls.
7. بناء القوالب والهوية البصرية.
8. إنشاء migrations وبيانات أولية.
9. إضافة Docker Compose وrequirements.
10. إضافة الاختبارات وتشغيل `check` و`test`.

## 19. معايير القبول النهائية
المشروع يعمل بـ Django SSR، كل الصفحات الأساسية تعرض HTML، القوالب منظمة، الهوية في ملف واحد، التطبيقات أنشئت بـ `startapp`, توجد migrations وبيانات أولية واختبارات، يعمل `python manage.py check`, Docker Compose يشغل Django وPostgreSQL، لوحة الإدارة المخصصة على `/admin/`، ولوحة Django الرسمية على `/django-admin/`.
