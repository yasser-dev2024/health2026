# تقرير طريقة البناء والبنية التحتية لمشروع صيف وصحة - مساعد

## 1. نظرة عامة على المشروع
مشروع **صيف وصحة - مساعد** هو منصة صحية وتوعوية عربية موجهة لزوار منطقة عسير. تم بناء المشروع باستخدام Django بأسلوب Server-Side Rendering، بحيث يتم توليد صفحات HTML من الخادم مباشرة عبر Django Templates، وتعمل الصفحات الأساسية حتى بدون الاعتماد على JavaScript.

تهدف المنصة إلى مساعدة الزائر على:
- إنشاء خطة صحية يومية حسب العمر والوجهة والغرض من الزيارة.
- معرفة الفعاليات الصحية والمواقع القريبة.
- تصفح الرسائل والمواد التوعوية.
- استخدام مساعد توعوي باسم الدكتور مساعد.
- مسح QR للمواقع والعناصر.
- جمع النقاط والأختام داخل جواز صحة عسير.
- المشاركة في تحدي بطل الصحة.

## 2. طريقة البناء البرمجية
تم بناء المشروع كتطبيق Django منظم إلى عدة تطبيقات مستقلة، بحيث يكون لكل جزء من النظام مسؤولية واضحة. هذا التقسيم يسهل الصيانة والتوسع ويمنع وضع كل المنطق في تطبيق واحد.

التطبيقات الأساسية:
- `core`: الصفحة الرئيسية، الإعدادات العامة، الهوية، الميزات، والدخول الذكي.
- `journey`: نموذج الرحلة وخدمة إنشاء الخطة الصحية اليومية.
- `events`: إدارة وعرض الفعاليات الصحية.
- `locations`: إدارة وعرض المواقع الصحية والقريبة.
- `awareness`: الرسائل التوعوية ومكتبة التحميلات.
- `assistant`: الدكتور مساعد والكلمات المفتاحية والإجابات.
- `passport`: جواز صحة عسير، الأختام، النقاط، والإنجازات.
- `qr`: مواقع QR، عناصر QR، تسجيل المسح، وتوليد QR للطباعة.
- `hero`: تحدي بطل الصحة والأسئلة والنتائج.
- `dashboard`: لوحة الإدارة المخصصة وسجل العمليات.

## 3. نمط العرض Frontend
تم اعتماد Django Templates بدلا من بناء واجهة منفصلة. كل صفحة ترث من قالب أساسي `base.html`، مع تضمين ملفات مشتركة مثل الهيدر، الفوتر، شاشة السبلاش، شاشة الدخول الصحي الذكي، والتنقل السفلي للجوال.

هيكلة القوالب:
- `templates/base.html`
- `templates/header.html`
- `templates/footer.html`
- `templates/splash.html`
- `templates/smart_entry.html`
- `templates/mobile_nav.html`
- مجلد خاص لكل تطبيق داخل `templates/`

تم تصميم الواجهة باتجاه RTL وبهوية بصرية مستوحاة من مشروع تجمع عسير الصحي القديم، مع دعم عرض الجوال عبر:
- شاشة Splash متجاوبة.
- شاشة دخول صحي ذكي بعد السبلاش.
- شريط تنقل سفلي للجوال.
- أزرار عائمة للدكتور مساعد وخدمة "أنا الآن".

## 5. قاعدة البيانات
يدعم المشروع PostgreSQL كقاعدة بيانات أساسية عند التشغيل عبر Docker، مع استخدام SQLite محليا لتسهيل التطوير والفحص السريع.

في البيئة المحلية:
- يستخدم المشروع SQLite تلقائيا إذا لم يتم ضبط متغير `DB_ENGINE`.

في بيئة Docker:
- يستخدم المشروع PostgreSQL عبر خدمة `db`.
- يتم تشغيل Django عبر خدمة `web`.

أهم الجداول/النماذج:
- إعدادات الصفحة الرئيسية.
- الفعاليات الصحية.
- المواقع الصحية.
- الرسائل والمحتوى التوعوي.
- أسئلة الدكتور مساعد والكلمات المفتاحية.
- جواز الزائر والأختام والإنجازات.
- مواقع وعناصر QR وسجل المسح.
- أسئلة ونتائج تحدي بطل الصحة.
- سجل العمليات الإدارية.

## 6. البنية التحتية للتشغيل
تم تجهيز المشروع للتشغيل بطريقتين:

### التشغيل المحلي
يتم التشغيل باستخدام البيئة الافتراضية الموجودة داخل المشروع:

```powershell
.\venv\Scripts\python.exe manage.py migrate
.\venv\Scripts\python.exe manage.py runserver
```

بعد التشغيل يكون الرابط:

`http://127.0.0.1:8000/`

### التشغيل عبر Docker Compose
تم تجهيز ملف `docker-compose.yml` لتشغيل:
- خدمة PostgreSQL.
- خدمة Django web.

الأمر:

```powershell
docker compose up --build
```

يقوم Docker Compose بتشغيل قاعدة البيانات ثم تنفيذ migrations ثم تشغيل خادم Django.

## 9. منطق QR
يدعم النظام نوعين من QR:
- QR للمواقع.
- QR للعناصر.

عند مسح QR يتم تسجيل:
- معرف الزائر `visitor_id`.
- نوع QR.
- المسار.
- الوقت.
- الموقع أو العنصر المرتبط.

كما يدعم النظام روابط QR القديمة بالشكل:

`/?qr=slug&ql=1&qrName=...&v=9`

وتقوم لوحة الإدارة بتوليد QR PNG قابل للطباعة باللون الرسمي للمشروع.

إذا كان QR مرتبطا بختم، يتم منح الختم للزائر مرة واحدة فقط، ثم تحديث النقاط في جواز صحة عسير.

## 10. جواز صحة عسير
يعتمد الجواز على معرف زائر عشوائي محفوظ في الجلسة، ولا يتم حفظ رقم الجوال أو موقع دقيق.

الجواز يحتوي على:
- النقاط.
- الأختام.
- الإنجازات.
- سجل زيارات QR.

القواعد:
- لا يتم منح نفس الختم لنفس الزائر أكثر من مرة.
- النقاط تزيد حسب الختم.
- الإنجازات تفتح عند تحقيق شروط معينة مثل عدد الأختام أو النقاط.

## 11. الدكتور مساعد
الدكتور مساعد هو محرك توعوي بسيط يعتمد على الكلمات المفتاحية العربية. يقوم بتطبيع النص العربي قبل البحث، مثل:
- توحيد الهمزات.
- إزالة التشكيل.
- تحويل التاء المربوطة.
- توحيد المسافات.

عند اكتشاف كلمات طارئة مثل ألم صدر أو اختناق أو نزيف، يعرض النظام رسالة طوارئ تتضمن:
- 997
- 937

## 12. الأمان والصلاحيات

- يوجد سجل عمليات إدارية `OperationLog`.
